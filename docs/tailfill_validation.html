Modem 2x — tail-fill validation

Branch feat/modem-2x, HEAD 603a80e. Validated 2026-05-19 on Pi5.

1. Context

Before this change, a HIGH+2X 10 kB burst at 30 % RaptorQ repair emitted 61 data CWs across 7 PLHEADER cycles, truncating the last cycle when n_total % cw_per_cycle != 0. The TX inserted INTER_FRAME_SILENCE_S = 0.2 s of audio silence between the data and the EOT PLHEADER. The RX (rx_v4.rs:1808-1828) detected the silence via an energy-gate and exited the cycle decode loop early, leaving CW 61 and 62 unconverged. The payload was still recovered byte-exact via RaptorQ repair margin, but 2 CW of bandwidth were wasted every burst.

Per [[feedback-tail-pad-useful-content]], the wire must carry useful content end-to-end : no silence in mid-burst, no zero-padding inside RaptorQ source packets. Three commits land the change end-to-end.

  • Profile : HIGH+2X (32-APSK 3/4, sps 10, Rs 3000 Bd, audio 48 kHz)
  • Payload : 10 000 bytes random
  • RaptorQ repair : --repair-pct 30 (default)

  • Drift estimator : locked en dur via RX2X_LOCK_DRIFT_PPM=<value> (commit a7b8e11) to isolate the resampler from the open estimator bug ([[rx2x-non-streaming-leak]]).

Triple confirmation : with the drift estimator locked, the polyphase resampler + worker cpal-chunks chain holds at ±300 ppm without degradation. Tail-fill brings all 70 emitted CWs to convergence — the 2 CW deficit was structural (silence-gate), not a noise floor.

4.1 Numerical breakdown

  • Pre-tail-fill : k_src=47 + n_repair=14 = 61 packets on the wire, 2 silence slots.
  • Tail-fill : n_emit = round_up(61, cw_per_cycle=10) = 70 packets, 0 silence in-cycle.
  • Burst duration : +1 cycle = +2.96 s (~11 % longer wire time, 9 extra repair CWs).
  • σ²(pilot) improvement : 0.00154 → 0.00100 (+1.9 dB). Plausibly the inter-frame PRBS (continuous LFSR-15, replacing the 200 ms silence) keeps the FFE adapted longer ; or just a sample-count artifact from the longer burst.

5. Results — estimator NOT locked (orthogonal bug)

Same TX (tail-fill HEAD) with RX2X_LOCK_DRIFT_PPM unset @ static +130 ppm :

HIGH+2X     0.10  0.00   0.04556   13.4    0.03447      -/30  10020     N  10.2

σ²=0.046, 0/30 CW converged, decode non-exact. The drift estimator's non-streaming bootstrap ([[rx2x-non-streaming-leak]]) commits an incoherent ratio that wrecks the symbol stream. Tail-fill is orthogonal to this bug — both must be fixed for the end-to-end pipeline to be production-ready.
